Manage your projects

Save quickly. Edit directly.

Project Manager starts with a simple model: any folder or workspace can become a named project. When you need more control, edit the underlying projects file directly inside VS Code.

Save Project

Turn the current folder or workspace into a favorite with a suggested name and a lightweight command-driven flow.

Edit Projects

Customize names, paths, tags, enabled state, and profiles in the projects file. Home shortcuts like ~ and $home are supported.

Access patterns

Open what you need, how you need it.

The extension covers both fast switching and deliberate context management. Open projects in the current window, force a new window, or drive the picker entirely from the keyboard.

Available commands

  • Save Project
  • Edit Project
  • List Projects to Open
  • List Projects to Open in New Window
  • Filter Projects by Tag

Keyboard focused users

Use the inProjectManagerList context key to wire Vim-style navigation or other custom movement commands into the picker.

Configuration

Control sorting, detection, grouping, and display.

Project list behavior

  • Sort by saved order, name, path, or recent usage
  • Group favorites and detected repositories by type
  • Optionally remove the current project from the list
  • Show parent folder info for duplicate names

Detection controls

  • Search configured base folders or glob patterns
  • Ignore folders like node_modules, out, or custom matches
  • Limit recursion depth and ignore nested projects if needed
  • Cache detected projects between sessions for faster lists

Status bar and windows

  • Show the project name in the status bar
  • Open in a new window from status bar interactions
  • Choose when an empty window should be reused
  • Highlight the current project in the side bar

Tags and storage

  • Define a shared tag vocabulary such as Personal, Work, or Learning
  • Control whether tag groups start expanded or collapsed
  • Store the projects file in a custom location
  • Use home path shortcuts for portable configuration

Remote development

Works locally by default, scales to remote-first setups.

Local machine, occasional remotes

This is the default path. Install locally and save Containers, SSH targets, WSL projects, or Codespaces favorites alongside local work. The extension opens the right remote automatically.

Mostly remote work

If your repositories live mainly on remote targets, run the extension there too. The README recommends setting remote.extensionKind so Project Manager can store and discover projects in the remote environment.

Side Bar

A dedicated view for projects and tags.

Beyond the quick pick, Project Manager ships a full side bar experience for browsing favorites, seeing the current project, and filtering work through project tags.
